# MOTIVATION.md

## Исследовательский вопрос

**Насколько хорошо модель классификации радиоизображений СРГ, обученная на данных 3000 МГц, обобщается на данные других частотных каналов (6000 МГц и 12200 МГц)?**

Формально: пусть $f_{3000}$ — модель классификации, обученная на множестве радиоизображений $\mathcal{D}_{3000} = \{(X_i^{(3000)}, y_i)\}_{i=1}^N$, где $X_i^{(3000)}$ — радиоизображение на частоте 3000 МГц, а $y_i \in \{1, \ldots, K\}$ — метка класса (тип солнечной активности).

Рассмотрим два дополнительных множества данных, полученных в близкие моменты времени:
- $\mathcal{D}_{6000} = \{(X_i^{(6000)}, y_i)\}$ — изображения на частоте 6000 МГц,
- $\mathcal{D}_{12200} = \{(X_i^{(12200)}, y_i)\}$ — изображения на частоте 12200 МГц,

при условии $\left| t_i^{(3000)} - t_i^{(6000)} \right| \le 3\ \text{мин}$ и $\left| t_i^{(3000)} - t_i^{(12200)} \right| \le 3\ \text{мин}$.

**Гипотеза:** Если $f_{3000}$ даёт согласованные предсказания на парах $(X_i^{(3000)}, X_i^{(6000)})$ и $(X_i^{(3000)}, X_i^{(12200)})$, то дообучение отдельных моделей на $\mathcal{D}_{6000}$ и $\mathcal{D}_{12200}$ не требуется, что позволяет сэкономить вычислительные ресурсы и время разработки.

**Вопросы, на которые мы отвечаем:**

1. Какова **эмпирическая доля совпадений** предсказаний $f_{3000}$ на парах каналов (3000↔6000 МГц и 3000↔12200 МГц) при $N = 1000$ случайно выбранных временных окнах?
2. Зависит ли качество согласования от **класса солнечной активности**? Есть ли классы, для которых модель «переносится» хуже?
3. Достаточно ли **статистической мощности** эксперимента (1000 испытаний) для принятия решения о пригодности модели к кросс-частотному инференсу?
4. Каковы **практические последствия**: можно ли использовать одну модель для оперативного прогнозирования по всем трём каналам СРГ?

## Формат работы

Эксперимент **уже выполнен** — исходный прогон занял ~73 мин (3000 запросов к API классификации). Данные собраны в `logs/` в формате JSONL. Для воспроизведения результатов не требуется:

- скачивать файлы с FTP-сервера СРГ,
- выполнять HTTP-запросы к API инференса,
- иметь доступ к исходным `.fit`-изображениям.

Вместо этого используется MCP-сервер `tools/experiment_mcp.py`, который предоставляет 5 инструментов для чтения логов и вычисления метрик:

| Инструмент | Назначение |
|-----------|-----------|
| `explore_data()` | Структура данных, каналы, количество триплетов, баланс классов |
| `explore_temporal()` | Помесячное распределение триплетов |
| `compute_metrics()` | Все метрики H1–H9 с доверительными интервалами и статистическими тестами |
| `generate_plots(output_dir?)` | 6 графиков: agreement_bar, distribution_by_channel, confidence_boxplot, agreement_by_class, agreement_vs_deltat, agreement_by_month |
| `get_h1_summary()` | Самосогласованность модели (H1) |

Время работы: ~1 мин вместо 73 мин API-запросов.

## Ограничения

- **Данные**: только 2025 год, три канала (3000 / 6000 / 12200 МГц), 1000 триплетов.
- **Исходные источники** (для справки, не для загрузки):
  ```
  FTP:   https://ftp.rao.istp.ac.ru/SRH/
  API:   https://forecasting.iszf.irk.ru/api/srh/predict
  ```
- **Предобработка**: временное рассогласование внутри триплета ≤ 3 мин; seed = 42 для воспроизводимости.
- **Анализ через MCP**: все метрики вычисляются по предсобранным логам; доступ к сырым данным — только через инструменты `explore_*`.
- **Статистика**:
  - оценка доли совпадений с бутстрэп-доверительными интервалами (1000 ресэмплингов, seed=42),
  - поправка на множественные сравнения при анализе по классам (Бонферрони или Бенджамини–Хохберг).
