## [2025-06-15 10:00] Этап 2. Декомпозиция и матпостановка — блоки M.0–M.9

**Контекст:** На основе `statement/MOTIVATION.md` и `statement/LIT.md` выполнена декомпозиция задачи, выписаны обозначения, сформулированы предположения, задача разбита на логические блоки.

---

### M.0 — Постановка задачи

Проверить, насколько модель классификации $f_{3000}$, обученная на радиоизображениях СРГ на частоте 3000 МГц, сохраняет согласованность предсказаний на данных 6000 МГц и 12960 МГц при $\Delta t \le 3$ мин. Вопрос диктуется практической потребностью: можно ли использовать единую модель для всех частотных каналов без дообучения.

---

### M.1 — Обозначения

| Символ | Значение |
|--------|----------|
| $f_{3000}$ | Модель классификации (Ensemble CLIP+EfficientNet+CatBoost), обученная на 3000 МГц |
| $\mathcal{D}_{3000}$ | Множество пар (изображение, метка) на 3000 МГц |
| $\mathcal{D}_{6000}$ | То же на 6000 МГц |
| $\mathcal{D}_{12960}$ | То же на 12960 МГц |
| $X_i^{(\nu)}$ | $i$-е радиоизображение на частоте $\nu \in \{3000, 6000, 12960\}$ |
| $y_i$ | Истинная метка класса (совпадает для всех $\nu$ при $\Delta t \le 3$ мин) |
| $\hat{y}_i^{(\nu)} = f_{3000}(X_i^{(\nu)})$ | Предсказание модели для $X_i^{(\nu)}$ |
| $K$ | Число классов (GOOD/BAD — бинарная классификация) |
| $t_i^{(\nu)}$ | Временна́я метка снимка |
| $\Delta t = \max_{\nu_1,\nu_2} \|t_i^{(\nu_1)} - t_i^{(\nu_2)}\|$ | Максимальное расхождение между снимками тройки |
| $N = 1000$ | Число испытаний |
| $S_{ab} = \frac{1}{N}\sum_{i=1}^N \mathbb{I}[\hat{y}_i^{(a)} = \hat{y}_i^{(b)}]$ | Эмпирическая доля совпадений предсказаний между каналами $a$ и $b$ |
| $\text{API}$ | Эндпоинт `POST https://forecasting.iszf.irk.ru/api/srh/predict` |
| $\text{seed} = 42$ | Фиксированный seed для воспроизводимости |

---

### M.2 — Предположения

1. **Физическая когерентность:** при $\Delta t \le 3$ мин класс $y_i$ един для всех трёх частот (солнечная активность не успевает существенно измениться).
2. **Инвариантность формата:** API принимает `.fit`-файлы с любого канала; предобработка (нормализация, логарифмирование) едина для всех частот.
3. **Детерминированность модели:** $f_{3000}$ детерминирована (одинаковый вход → одинаковый выход).
4. **Качество данных:** только изображения, прошедшие CLEAN-калибровку (cleanMaps).
5. **Доступность данных:** для любого случайного дня в 2025 году есть снимки на всех трёх частотах с $\Delta t \le 3$ мин (проверяется эмпирически).

---

### M.3 — Мера согласованности

Основная метрика — доля совпадений предсказаний $S_{ab}$ для двух пар:
- $S_{3000,6000}$ — обученная модель vs данные 6000 МГц
- $S_{3000,12960}$ — обученная модель vs данные 12960 МГц

Дополнительно:
- Bootstrap 95%-доверительный интервал для $S_{ab}$ (1000 ресэмплингов)
- $\chi^2$-тест на независимость согласованности от частотного канала (H₀: $S_{3000,6000} = S_{3000,12960}$)

---

### M.4 — Сбор данных (критерии отбора временных окон)

1. Из 2025 года равномерно выбрать $N = 1000$ случайных моментов времени.
2. Для каждого момента найти ближайшие cleanMaps на FTP:
   - `SRH0306/cleanMaps/...` для 3000 МГц
   - `SRH0612/cleanMaps/...` для 6000 МГц
   - `SRH1224/cleanMaps/...` для 12960 МГц
3. Отфильтровать тройки с $\Delta t > 3$ мин.
4. Скачать `.fit`-файлы, отправить в API.

---

### M.5 — Инференс

```
POST /api/srh/predict
Content-Type: multipart/form-data
File: X_i^{(ν)}.fit

Response:
{
    "label": "GOOD" | "BAD",
    "probability": float,
    "image": base64
}
```

Лимит: 1 запрос/сек. Логирование всех запросов/ответов в `logs/inference_{timestamp}.jsonl`.

---

### M.6 — Статистический анализ

1. Точечная оценка $S_{ab}$ и bootstrap CI (1000 ресэмплингов).
2. $\chi^2$-тест (или точный тест Фишера) для сравнения $S_{3000,6000}$ и $S_{3000,12960}$.
3. Стратификация по классам: $S_{ab}^{(GOOD)}$ и $S_{ab}^{(BAD)}$.
4. Поправка Бонферрони при стратифицированных сравнениях.

---

### M.7 — Визуализация

1. Гистограмма совпадений по парам каналов с CI.
2. Stacked bar plot: канал × класс (GOOD/BAD).
3. Confusion matrices: $f_{3000}$ на 3000 vs 6000 vs 12960.

Графики сохраняются в `experiment/plots/`.

---

### M.8 — Практический критерий пригодности

Модель считается пригодной для кросс-частотного инференса, если:
- $S_{3000,6000} \ge 0.90$ и $S_{3000,12960} \ge 0.85$ (пороги обсуждаемы)
- Или разница $|S_{3000,6000} - S_{3000,12960}| < 0.05$ и незначима по $\chi^2$-тесту ($p > 0.05$)

В противном случае рекомендуется дообучение или отдельная модель для старших частот.

---

### M.9 — Ограничения метода

- Оценка согласованности — **не** оценка точности: мы не знаем истинную метку $y_i$.
- Данные только за 2025 год.
- Три частотных канала: экстраполяция на другие частоты (8–24 ГГц) не проверяется.
- Возможны систематические ошибки FTP (пропуски, битые файлы).

## [2026-06-15 15:35] Этап 5. Реализация и проведение эксперимента

**Контекст:** Написание experiment.py, отладка FTP-клиента, сбор 1000 триплетов (3000+6000+12200 МГц × 1000 = 3000 API-запросов), проверка гипотез H1–H9.

### Проблемы, выявленные в процессе:
1. **FTP-client**: паттерны имён файлов не совпадали с фактическими (srh_I_YYYY-MM-DD vs srh_YYYYMMDD_T...); исправлено через list_files по формату SRH{code}/cleanMaps/YYMMDD/
2. **Label mismatch**: API возвращает метки "Ok"/"Bad" (не "GOOD"/"BAD" как ожидалось); исправлено динамическое определение label-строк в коде H4/H7
3. **Pool bias**: первоначальный сбор триплетов брал только январь (max_triplets=1000 достигнут на 20-м дне); исправлено до равномерной стратификации 100/месяц
4. **Timeout**: 1-часовой лимит инструмента не покрывал ~73 мин выполнения (3000 запросов × 1 сек); решено увеличением таймаута до 2 ч

### Ключевые результаты:
| Гипотеза | Результат | Ключевая метрика |
|----------|-----------|-------------------|
| H1 (самосогласованность) | ✅ PASS | S_self=1.0000 [1.0, 1.0] |
| H2 (симметрия) | ❌ ОТКЛОНЕНА | S_6=0.671, S_12=0.542, p=3.5e-9 |
| H3 (пороги) | ❌ FAIL | 6000<0.90, 12200<0.85 |
| H4 (стратификация) | ✅ ВАЖНО | Bad: 79.8%/91.9% vs Ok: 56.4%/22.6% |
| H5 (уверенность) | ✅ ПАДЕНИЕ | 0.932→0.922→0.893, p=7.4e-12 |
| H6 (Δt) | ❌ НЕТ | ρ≈0.056/0.024 |
| H7 (распределения) | ❌ РАЗНЫ | Ok rate: 54%→40%→16% |
| H8 (сезонность) | ❌ НЕСТАБИЛЬНА | 36.7%–100%, p=3.6e-28 |
| H9 (baseline) | ✅ ЛУЧШЕ | 0.671 vs 0.399, p≈0 |

### Главный вывод:
Модель f_{3000} не обобщается на другие частоты с практической точностью. Согласованность с 6000 МГц = 67.1% (нужно ≥90%), с 12200 МГц = 54.2% (нужно ≥85%). Причина: дрейф распределения — на 12200 МГц модель предсказывает "Bad" в 84% случаев (vs 54% на 3000). При этом модель хорошо обобщается для "Bad"-класса (91.9% совпадений 3000↔12200), но плохо для "Ok" (22.6%).
