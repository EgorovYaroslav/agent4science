## 3. Что ты ОБЯЗАН делать

### 3.1. Вести STATUS.md

После **каждого блока работы** (перед тем как закончить сессию) обновляй `statement/STATUS.md`:

```markdown
# STATUS.md

## Текущий этап
[Например: "Этап 2. Численный эксперимент"]

## Что сделано за последнюю сессию

## Следующие шаги
[план на следующую сессию]

## Проблемы/блокировки
[если есть]
```

**Без STATUS.md после разрыва SSH-сессии ты забываешь контекст!**

### 3.2. Вести diary.md (append-only)

Каждое действие записывай в `statement/diary.md` в формате:

```markdown
## [YYYY-MM-DD HH:MM] Краткое описание действия

**Контекст:** почему это делается
**Результат:** что получилось
**Файлы:** какие файлы созданы/изменены
```

### 3.3. Делать git-снэпшоты

Перед длинными экспериментами:
```bash
git add .
git commit -m "checkpoint: описание состояния"
git tag snapshot_$(date +%Y%m%d_%H%M%S)
```

Используй **линейную историю** (никаких `git rebase`, `git push --force`).

## 7. Конкретная задача: кросс-частотное обобщение СРГ

### Входные данные

**Вопрос:** Насколько хорошо модель классификации радиоизображений СРГ,
обученная на данных 3000 МГц, обобщается на данные других частотных
каналов (6000 МГц и 12200 МГц)?

**Данные уже собраны** — эксперимент выполнен, логи лежат в `logs/`:
- `logs/h1_*.jsonl` — 100 записей самосогласованности (H1)
- `logs/main_*.jsonl` — 3000 записей (1000 триплетов × 3 канала)

Структура записи в main-логе:
```json
{
  "trial": 0,
  "phase": "main",
  "frequency": 3000,
  "url": "https://...fit",
  "dt": "2025-03-01 05:59:27",
  "label": "Ok",
  "probability": 0.996,
  "max_dt_sec": 54.0
}
```

**Исходные источники (для справки, не для загрузки):**
- FTP: https://ftp.rao.istp.ac.ru/SRH/ (данные СРГ за 2025 год)
- API: https://forecasting.iszf.irk.ru/api/srh/predict

### MCP-инструменты

Вся работа с данными — через MCP-сервер `tools/experiment_mcp.py` (запускается автоматически из `opencode.json`):

| Инструмент | Что даёт |
|-----------|----------|
| `explore_data()` | Каналы, число триплетов, баланс классов, временной диапазон, схему данных |
| `explore_temporal()` | Распределение триплетов по месяцам |
| `compute_metrics()` | Все метрики H1–H9: согласованность, bootstrap CI, χ², t-test, Spearman, ANOVA, baseline |
| `generate_plots(output_dir?)` | 6 графиков: agreement_bar, distribution_by_channel, confidence_boxplot, agreement_by_class, agreement_vs_deltat, agreement_by_month |
| `get_h1_summary()` | H1 самосогласованность (S_self, CI) |

**Никаких HTTP-запросов к API, никаких FTP-загрузок.** Данные уже предобработаны.

### Исследовательские вопросы

1. Какова эмпирическая доля совпадений S предсказаний на парах (3000↔6000) и (3000↔12200)?
2. Зависит ли согласованность от класса (Ok vs Bad)?
3. Падает ли уверенность модели с ростом частоты?
4. Есть ли корреляция согласованности с временным рассогласованием Δt?
5. Стабильна ли согласованность по месяцам?
6. Превышает ли модель majority baseline?

### Гипотезы H1–H9

Проверяются вызовом `compute_metrics()`:

| # | Гипотеза | Ожидание |
|---|----------|----------|
| H1 | Самосогласованность ≥ 0.95 | S_self ≥ 0.95 |
| H2 | S(3000,6000) = S(3000,12200) | p ≥ 0.05 |
| H3 | S ≥ 0.90 (6000) и S ≥ 0.85 (12200) | оба порога |
| H4 | Асимметрия Ok vs Bad | разные S |
| H5 | Уверенность падает с частотой | p < 0.05 |
| H6 | Корреляция с Δt | ρ ≠ 0 |
| H7 | Распределения меток однородны | p ≥ 0.05 |
| H8 | Сезонная стабильность | p ≥ 0.05 |
| H9 | Модель > majority baseline | p < 0.05 |
